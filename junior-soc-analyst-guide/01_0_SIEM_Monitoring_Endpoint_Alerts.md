# Monitor Alerts from SIEM and Endpoint Tools

## 📋 Review Points

- [ ] Understand SIEM dashboard layout and navigation
- [ ] Know the different alert severity levels (Critical, High, Medium, Low, Info)
- [ ] Identify key metrics and KPIs in alert monitoring
- [ ] Recognize common alert sources (endpoint detection, network, log aggregation)
- [ ] Understand alert thresholds and triggers
- [ ] Know how to filter and search alerts effectively
- [ ] Understand false positives vs. true positives
- [ ] Know escalation criteria for different alert types

---

## 🔍 Step-by-Step Guide

### Step 1: Access and Navigate SIEM/Endpoint Tools
1. Log into your organization's SIEM platform (e.g., Splunk, ELK Stack, ArcSight)
2. Review the main dashboard for alert overview
3. Check the alert queue or inbox
4. Familiarize yourself with the toolbar and navigation menus
5. Note the current time zone and system time

### Step 2: Understand Alert Classifications
1. Identify the alert classification system used
2. Learn what each severity level means:
   - **Critical**: Immediate threat to systems/data
   - **High**: Significant security concern
   - **Medium**: Worth investigating
   - **Low**: Minor issues
   - **Info**: Informational only
3. Understand alert categories (malware, anomaly, intrusion, etc.)

### Step 3: Monitor for New Alerts
1. Set up alert filters for your assigned systems
2. Configure alert refresh intervals (typically 5-15 minutes)
3. Check alert trends and patterns
4. Monitor alert count over time
5. Identify peak alert times

### Step 4: Correlate Alerts
1. Look for related alerts from same source/user
2. Check timestamp alignment across tools
3. Identify alert chains that may indicate progression
4. Correlate endpoint alerts with network events
5. Document the relationship between alerts

### Step 5: Log and Document
1. Record alert details in incident tracking system
2. Note timestamp of first observation
3. Document alert source and event ID
4. Capture affected systems/users
5. Create initial incident record

## 🎯 Practice Activities

### Activity 1: Dashboard Familiarization (30 minutes)
1. Log into your SIEM system
2. Spend 15 minutes exploring all menus and tabs
3. Create 3 custom views for different alert types
4. Practice filtering by severity, time range, and source
5. Document the layout and key features

### Activity 2: Alert Classification Exercise (45 minutes)
1. Review 10-15 sample alerts provided by your team
2. Classify each by severity level
3. Explain your reasoning for each classification
4. Compare with actual classifications
5. Document learning points

### Activity 3: Alert Correlation Drill (1 hour)
1. Work with synthetic alert data set
2. Identify related alerts from a sequence
3. Document the timeline of events
4. Create a visual representation of alert relationships
5. Present findings to your team lead

### Activity 4: Metric Tracking Exercise (Ongoing)
1. Track alert volume daily for 1 week
2. Create a simple spreadsheet with daily counts
3. Identify patterns (peak times, days, alert types)
4. Document any anomalies noticed
5. Present findings in team meeting

### Activity 5: SIEM Tool Deep Dive (2-3 hours)
1. Complete vendor-provided SIEM training modules
2. Practice creating custom alerts/rules
3. Learn advanced search syntax
4. Build a dashboard for your assigned systems
5. Demonstrate to your team lead
