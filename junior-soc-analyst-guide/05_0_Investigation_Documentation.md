# Document Investigations

## 📋 Review Points

- [ ] Understand why documentation is critical
- [ ] Know the documentation standards and templates
- [ ] Understand what information to document
- [ ] Know how to organize information chronologically
- [ ] Understand evidence preservation requirements
- [ ] Know how to write clear, concise investigations
- [ ] Understand legal and compliance documentation needs
- [ ] Know how to secure investigation documentation

---

## 🔍 Step-by-Step Guide

### Step 1: Understand Documentation Requirements
1. Review your organization's documentation standards
2. Understand what goes into an incident record
3. Learn the required documentation format
4. Understand retention requirements
5. Learn about evidence preservation needs
6. Understand chain of custody requirements
7. Know what is discoverable vs. internal notes
8. Understand legal/compliance documentation needs

### Step 2: Create Initial Incident Record
1. Open your incident/ticketing system
2. Create new incident ticket immediately upon discovery
3. Record incident ID number
4. Document discovery timestamp
5. Identify and list the discoverer
6. Provide initial description of incident
7. Document initial assessment/severity
8. Note any immediate actions taken
9. Add appropriate labels/tags
10. Assign to appropriate team

### Step 3: Document Initial Findings
1. Record all relevant alert data
2. Document system names and IP addresses
3. Record user accounts involved
4. Document file paths, file names, file hashes
5. Record command lines and processes
6. Document network connections (source/destination/ports)
7. Include DNS queries and resolutions
8. Record time stamps for all events
9. Document external IPs/domains involved
10. Preserve screenshots or exports

### Step 4: Create Investigation Timeline
1. Start with earliest event detected
2. List events in chronological order
3. Include exact timestamps for each event
4. Document source of each time entry (log name, system)
5. Add context/description for each event
6. Note gaps in data or missing logs
7. Update timeline as investigation progresses
8. Use consistent timestamp format (UTC preferred)
9. Highlight critical events
10. Show progression of attack/incident

### Step 5: Document Investigation Steps
1. Record each analysis step performed
2. Note what tools were used
3. Document what was checked (yes/no/findings)
4. Record analyst name and timestamp for each step
5. Document methodical approach to investigation
6. Show thought process and reasoning
7. Note questions asked and answered
8. Document false leads investigated
9. Record dead ends and why they were eliminated
10. Show peer reviews or validations

### Step 6: Preserve Evidence
1. Export log entries related to incident
2. Capture screenshots of relevant findings
3. Document chain of custody:
   - Who accessed the evidence
   - When they accessed it
   - What they did with it
4. Create forensic images if needed
5. Hash critical files for integrity
6. Secure evidence in appropriate location
7. Document storage location
8. Create backups of evidence
9. Maintain access logs
10. Ensure evidence is immutable

### Step 7: Document Findings and Analysis
1. Create findings section in incident record
2. Clearly state what was found
3. Distinguish between facts and inferences
4. Support conclusions with evidence
5. Document confidence levels
6. Note any alternative explanations considered
7. Explain why certain explanations were ruled out
8. Include relevant log excerpts
9. Attach screenshots and evidence exports
10. Reference specific sources for each finding

### Step 8: Document Impact Assessment
1. List all affected systems
2. Document affected user accounts
3. Assess potential data exposure
4. Evaluate business impact
5. Determine scope of incident
6. Document number of affected assets
7. Note severity level and justification
8. Record any actual damage/loss
9. Document recovery steps needed
10. Estimate time to resolution

### Step 9: Document Recommendations
1. List immediate remediation steps
2. Document containment actions
3. Recommend investigation next steps
4. Suggest preventive measures
5. Recommend security controls
6. Suggest monitoring enhancements
7. Recommend policy updates
8. Document training needs identified
9. Suggest system patches/updates
10. Recommend additional testing

### Step 10: Final Documentation and Sign-Off
1. Write clear executive summary
2. Ensure all sections are complete
3. Review for accuracy and completeness
4. Add appropriate classifications (if needed)
5. Document final incident closure
6. Include lessons learned
7. Sign off investigation
8. Archive documentation appropriately
9. Create post-incident summary
10. Schedule follow-up review

## 🎯 Practice Activities

### Activity 1: Documentation Format Review (30 minutes)
1. Review your organization's incident templates
2. Understand each section and field
3. Learn the required information for each field
4. Create a quick reference guide
5. Familiarize yourself with the system

### Activity 2: Timeline Creation Practice (1 hour)
1. Receive sample incident scenario
2. Extract all time-stamped events
3. Create chronological timeline
4. Add context for each event
5. Identify critical events
6. Practice with 3-5 scenarios

### Activity 3: Documentation Writing Exercise (1-2 hours)
1. Review well-documented incident samples
2. Analyze their structure and content
3. Practice writing clear, concise documentation
4. Draft investigation documentation for 2-3 scenarios
5. Get feedback from experienced team members
6. Refine writing style and clarity

### Activity 4: Evidence Preservation Exercise (1 hour)
1. Learn evidence preservation procedures
2. Practice collecting and securing evidence
3. Document chain of custody correctly
4. Create evidence inventory
5. Practice labeling and storing evidence

### Activity 5: Complete Incident Documentation (3-4 hours)
1. Receive realistic incident scenario
2. Create complete incident documentation including:
   - Initial assessment
   - Detailed findings
   - Timeline of events
   - Evidence inventory
   - Impact assessment
   - Recommendations
3. Review with team lead
4. Incorporate feedback
5. Practice on multiple scenarios

### Activity 6: Lessons Learned Writing (1 hour)
1. Review past incident lessons learned
2. Understand what makes good lessons learned
3. Practice identifying lessons from scenarios
4. Write clear, actionable lessons learned
5. Practice presenting lessons to team

### Activity 7: Documentation Audit (45 minutes)
1. Review your documentation for completeness
2. Ensure all required sections are present
3. Verify evidence is properly documented
4. Check timeline accuracy
5. Validate conclusions are supported
6. Check evidence references

## ✅ Success Criteria

- Create incident records immediately
- Document investigations clearly and completely
- Maintain chronological timelines
- Preserve evidence properly
- Support conclusions with evidence
- Use organization's documentation standards
- Follow chain of custody procedures
- Create actionable recommendations
- Produce documentation suitable for executive review
- Maintain documentation confidentiality appropriately
- Document investigations within SLA timeframes
- Ensure documentation is discoverable and searchable
